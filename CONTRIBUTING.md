# Contributing

Thanks for your interest in improving this project.

## Quick start

1. Fork the repo
2. Create a branch: `git checkout -b feat/your-change`
3. Make changes
4. Commit clearly: `git commit -m "feat: short description"`
5. Push and open a Pull Request

## Guidelines

- Keep changes focused and small
- Preserve visual style unless PR is specifically a redesign
- Do not commit secrets or local config
- Update README when behavior or structure changes

## Local check

Open `index.html` in browser and verify:
- Page loads without missing assets
- No broken local links
- Main sections render correctly on desktop/mobile
